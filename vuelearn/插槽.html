# 插槽内容

Vue 实现了一套内容分发的 API，将 <slot> 元素作为承载分发内容的出口。

它运行合成组件：

<navigation-link url="/profile">
  Your Profile
</navigation-link>
1
2
3

然后在 <navigation-link> 模板中：

<a
  v-bind:href="url"
  class="nav-link"
>
  <slot></slot>
</a>
1
2
3
4
5
6

当组件渲染的时候， <slot></slot> 将会被替换为 Your Profile 。插槽内可以包含任何模板代码：

<navigation-link url="/profile">
  <!-- 添加一个 Font Awesome 图标 -->
  <span class="fa fa-user"></span>
  Your Profile
</navigation-link>
1
2
3
4
5

甚至其它的组件：

<navigation-link url="/profile">
  <!-- 添加一个图标的组件 -->
  <font-awesome-icon name="user"></font-awesome-icon>
  Your Profile
</navigation-link>
1
2
3
4
5

如果 <navigation-link> 的 template 中没有包含一个 <slot> 元素，则该组件起始标签和结束标签之间的任何内容都会被抛弃。

# 编译作用域

父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的。

该插槽和模板的其它地方一样可以访问相同的实例 property，而不能访问 <navigation-link> 的作用域，例如 url 是访问不到的。

<navigation-link url="/profile">
  Clicking here will send you to: {{ url }}
  <!--
  这里的 `url` 会是 undefined，因为其 (指该插槽的) 内容是
  _传递给_ <navigation-link> 的而不是
  在 <navigation-link> 组件*内部*定义的。
  -->
</navigation-link>

# 后备内容（可以理解为默认内容）

后备内容就是在没有为插槽提供内容时，供渲染的。

将 Submit 作为后备内容，在没有提供内容时，会渲染这个默认值。

如果我们提供了内容，提供的内容将会被渲染，从而替代后备内容。

<button type="submit">
	<slot>Submit</slot>
</button>

# 具名插槽

有时我们会在一个组件中，使用多个插槽，为了方便在使用时将不同的内容放到对应的插槽， <slot> 有一个 attribute name ：

<div class="container">
  <header>
    <slot name="header"></slot>
  </header>
  <main>
    <slot></slot>
  </main>
  <footer>
    <slot name="footer"></slot>
  </footer>
</div>

一个不带 name 的 <slot> 出口会带有隐含的名字“default”。

在向具名插槽提供内容时，可以在 template 元素上使用 v-slot 命令

<base-layout>
  <template v-slot:header>
    <h1>Here might be a page title</h1>
  </template>

  <p>A paragraph for the main content.</p>
  <p>And another one.</p>

  <template v-slot:footer>
    <p>Here's some contact info</p>
  </template>
</base-layout>

# 作用域插槽

有时需要插槽访问子组件才有的数据，可以将子组件的内容作为 slot 的一个属性绑定上去：

<span>
  <slot v-bind:user="user">
    {{ user.lastName }}
  </slot>
</span>
1
2
3
4
5

绑定在 slot 上的 attribute 被称为 插槽 prop

<current-user>
  <template v-slot:default="slotProps">
    {{ slotProps.user.firstName }}
  </template>
</current-user>

2
3
4
5

我们选择将包含所有插槽 prop 的对象命名为 slotProps，但你也可以使用任意你喜欢的名字。

# 独占默认插槽的缩写语法

v-slot 不带参数时对应默认插槽。

但是默认插槽的缩写语法不能和具名插槽混用。

出现多个插槽时，要使用完整的插槽写法。

# 解构插槽 Prop

使用 解构 传入具体的 prop

<current-user v-slot="{ user }">
  {{ user.firstName }}
</current-user>
1
2
3

prop 冒号可以重命名：

<current-user v-slot="{ user : person }">
  {{ person.firstName }}
</current-user>
1
2
3

在 prop 是 undefined 时，可以使用提前定义的后备内容：

<current-user v-slot="{ user = {firstName: 'zhangsan'} }">
  {{ user.firstName }}
</current-user>
1
2
3

# 动态插槽名

动态参数指令可以用在 v-slot 上，定义动态的插槽名：

<base-layout>
  <template v-slot:[dynamicSlotName]>
    ...
  </template>
</base-layout>

# 具名插槽的缩写

参数之前的 v-slot: 缩写为 # ；例如 v-slot:header 缩写为 #header ：

<base-layout>
  <template #header>
    <h1>Here might be a page title</h1>
  </template>

  <p>A paragraph for the main content.</p>
  <p>And another one.</p>

  <template #footer>
    <p>Here's some contact info</p>
  </template>
</base-layout>
